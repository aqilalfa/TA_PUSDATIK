# BAB 4.5: QUALITY GATES DAN GUARDRAILS UNTUK SISTEM SAFETY

## 4.5.1 Input Validation dan Guardrails

### 4.5.1.1 Input Security Layer

Sebelum proses retrieval, query melalui multi-layer security validation:

```python
async def validate_and_sanitize_query(query: str) -> Tuple[bool, str]:
    """
    Validate query untuk security threats dan format correctness.
    
    Returns: (is_valid: bool, error_message_or_cleaned_query: str)
    """
    
    # 1. Length validation
    if not query or len(query.strip()) < 2:
        return False, "Query terlalu pendek"
    if len(query) > 5000:
        return False, "Query terlalu panjang (max 5000 characters)"
    
    # 2. Format validation
    if contains_sql_injection_pattern(query):
        return False, "Query mengandung pola SQL injection"
    
    # 3. Jailbreak detection
    try:
        guard = Guard().use(DetectJailbreak, threshold=0.9)
        guard.validate(query)
    except Exception as e:
        logger.warning(f"Jailbreak detected: {e}")
        return False, "Query terdeteksi sebagai potensi jailbreak"
    
    # 4. Malicious intent detection (pattern-based)
    malicious_patterns = [
        r"ignore all previous|forget|disregard",  # Prompt injection
        r"system prompt|instructions|rule",        # Attempt to access system
        r"execute|run.*script|code",              # Code injection
    ]
    
    for pattern in malicious_patterns:
        if re.search(pattern, query, re.IGNORECASE):
            return False, "Query mengandung pola yang mencurigakan"
    
    # 5. Sanitization
    cleaned_query = query.strip()
    cleaned_query = re.sub(r'\s+', ' ', cleaned_query)  # Normalize whitespace
    
    return True, cleaned_query
```

### 4.5.1.2 Dynamic Guardrail Generation

Guardrails di-generate secara dinamik berdasarkan query dan context:

```python
def build_guardrail_instructions(query: str, retrieved_context: str) -> str:
    """
    Build dynamic guardrails untuk improve answer quality.
    """
    
    guardrails = []
    
    # 1. Table query guardrail
    table_guardrail = build_table_guardrail(query, retrieved_context)
    if table_guardrail:
        guardrails.append(table_guardrail)
    
    # 2. Generic grounding guardrail
    grounding_guardrail = build_generic_grounding_guardrail(query, retrieved_context)
    if grounding_guardrail:
        guardrails.append(grounding_guardrail)
    
    # 3. List-completeness guardrail
    if re.search(r'\b(apa saja|sebutkan|daftar|rincian|langkah)\b', 
                 query, re.IGNORECASE):
        guardrails.append(
            "- Karena pertanyaan meminta rincian/daftar, berikan jawaban "
            "dalam format list lengkap dengan bullet points atau numbering."
        )
    
    # 4. Citation guardrail
    guardrails.append(
        "- Sertakan [n] citation untuk SETIAP kalimat yang informasinya "
        "berasal dari dokumen yang diberikan."
    )
    
    return "\n".join(guardrails)
```

## 4.5.2 Output Validation dan Quality Scoring

### 4.5.2.1 Answer Quality Assessment

Setelah LLM generate jawaban, dilakukan quality assessment:

```python
class AnswerQualityScorer:
    """Score answer quality berdasarkan multiple dimensions."""
    
    def score_answer(
        self,
        answer: str,
        query: str,
        context: str,
        query_type: str = "general"
    ) -> Dict[str, float]:
        """
        Score answer dan return detailed metrics.
        
        Returns: {
            'faithfulness': 0-1,      # Is answer grounded in context?
            'relevance': 0-1,         # Does answer address query?
            'completeness': 0-1,      # Is answer complete for query type?
            'hallucination': 0-1,     # Does answer contain hallucinations?
            'citation_coverage': 0-1, # Are sources cited?
            'overall_quality': 0-1    # Weighted average
        }
        """
        
        scores = {}
        
        # 1. Faithfulness: Check if claims are grounded in context
        scores['faithfulness'] = self._score_faithfulness(answer, context)
        
        # 2. Relevance: Does answer address the query?
        scores['relevance'] = self._score_relevance(answer, query)
        
        # 3. Completeness: Is answer complete for query type?
        scores['completeness'] = self._score_completeness(
            answer, query, query_type
        )
        
        # 4. Hallucination detection
        scores['hallucination'] = self._detect_hallucinations(answer, context)
        
        # 5. Citation coverage
        scores['citation_coverage'] = self._score_citation_coverage(answer)
        
        # 6. Weighted overall score
        weights = {
            'faithfulness': 0.35,
            'relevance': 0.25,
            'completeness': 0.15,
            'hallucination': 0.15,
            'citation_coverage': 0.10
        }
        
        scores['overall_quality'] = sum(
            scores[k] * weights[k] for k in weights
        )
        
        return scores
```

### 4.5.2.2 Hallucination Detection

Sistem mendeteksi several jenis hallucination:

```python
def detect_hallucinations(answer: str, context: str) -> float:
    """
    Detect hallucinations dalam answer.
    
    Returns: hallucination_score (0=no hallucination, 1=high hallucination risk)
    """
    
    hallucination_indicators = []
    
    # 1. Pasal/Ayat yang tidak ada di context
    pasal_pattern = re.compile(r'Pasal\s+(\d+)\s+Ayat\s+(\d+)', re.IGNORECASE)
    for match in pasal_pattern.finditer(answer):
        pasal_num = match.group(1)
        ayat_num = match.group(2)
        
        if not re.search(
            rf'Pasal\s+{pasal_num}.*?Ayat\s+{ayat_num}',
            context,
            re.DOTALL|re.IGNORECASE
        ):
            hallucination_indicators.append({
                'type': 'fake_pasal_ayat',
                'value': f'Pasal {pasal_num} Ayat {ayat_num}'
            })
    
    # 2. Tabel yang tidak ada
    table_pattern = re.compile(r'Tabel\s+(\d+)', re.IGNORECASE)
    for match in table_pattern.finditer(answer):
        table_num = match.group(1)
        
        if not re.search(rf'Tabel\s+{table_num}', context, re.IGNORECASE):
            hallucination_indicators.append({
                'type': 'fake_table',
                'value': f'Tabel {table_num}'
            })
    
    # 3. Claims explicitly marked as "tidak ditemukan" ketika info ada
    unavailable_claims = [
        "tidak tercantum",
        "tidak tersedia",
        "tidak ditemukan"
    ]
    
    for claim in unavailable_claims:
        if claim in answer.lower():
            # Check if related info exists in context
            query_terms = extract_query_terms(answer)
            context_lower = context.lower()
            
            if any(term in context_lower for term in query_terms):
                hallucination_indicators.append({
                    'type': 'false_unavailability_claim',
                    'claim': claim
                })
    
    # 4. Contradictions with context
    if has_contradictions_with_context(answer, context):
        hallucination_indicators.append({
            'type': 'context_contradiction',
            'severity': 'high'
        })
    
    # Score hallucination risk
    if not hallucination_indicators:
        return 0.0  # No hallucinations
    
    # Weight by severity
    hallucination_score = 0.0
    for indicator in hallucination_indicators:
        if indicator['type'] in ['fake_pasal_ayat', 'fake_table']:
            hallucination_score += 0.3  # Critical
        elif indicator['type'] == 'false_unavailability_claim':
            hallucination_score += 0.2  # Serious
        elif indicator['type'] == 'context_contradiction':
            hallucination_score += 0.25  # Critical
    
    return min(hallucination_score, 1.0)
```

## 4.5.3 Toxic Language dan Safety Filtering

### 4.5.3.1 Output Safety Validation

Jawaban di-validate untuk memastikan tidak mengandung konten berbahaya:

```python
async def validate_output_safety(answer: str, query_context: str) -> Tuple[bool, str]:
    """
    Validate output untuk toxic language, bias, dan ethical concerns.
    """
    
    # 1. Toxic language detection
    try:
        guard = Guard().use(
            ToxicLanguage,
            threshold=0.5,
            validation_method="sentence",
            on_fail="exception"
        )
        guard.validate(answer)
    except Exception as e:
        logger.warning(f"Toxic language detected: {e}")
        return False, "Jawaban mengandung konten yang tidak sesuai"
    
    # 2. Bias detection
    bias_patterns = [
        r'\b(seharusnya|harus|wajib)',  # Prescriptive language
        r'(?:semua|selalu|tidak pernah)',  # Generalization
    ]
    
    # Context-dependent check
    if query_context == "legal" and any(
        re.search(p, answer, re.IGNORECASE) for p in bias_patterns
    ):
        # In legal domain, overly prescriptive language is concerning
        logger.warning("Potentially biased language in legal answer")
        # Not blocking, just logging
    
    # 3. Information exposure check
    sensitive_patterns = [
        r'\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b',  # Email
        r'\b\d{3}-\d{2}-\d{4}\b',                                 # SSN-like
        r'\b\d{16}\b',                                            # Credit card-like
    ]
    
    if any(re.search(p, answer) for p in sensitive_patterns):
        logger.warning("Potential sensitive data exposure")
        return False, "Jawaban mengandung informasi sensitif"
    
    return True, ""
```

### 4.5.3.2 Quality Threshold Gates

System memiliki hard threshold untuk quality metrics:

```python
class QualityThresholdGates:
    """Gates that block low-quality answers."""
    
    # Thresholds (dapat di-tune berdasarkan domain)
    FAITHFULNESS_THRESHOLD = 0.75      # Must be grounded in context
    RELEVANCE_THRESHOLD = 0.70         # Must answer the question
    HALLUCINATION_THRESHOLD = 0.10     # Low tolerance untuk hallucination
    CITATION_COVERAGE_THRESHOLD = 0.80 # Legal domain: high citation requirement
    OVERALL_QUALITY_THRESHOLD = 0.72   # Overall quality score
    
    def check_quality_gates(
        self,
        scores: Dict[str, float],
        query_type: str
    ) -> Tuple[bool, List[str]]:
        """
        Check if answer passes quality gates.
        
        Returns: (passed: bool, reasons_if_failed: List[str])
        """
        
        failures = []
        
        # Adjust threshold berdasarkan query type
        if query_type == "pasal":
            # Legal quotes require higher standards
            self.CITATION_COVERAGE_THRESHOLD = 0.95
            self.FAITHFULNESS_THRESHOLD = 0.90
        
        # Check each gate
        if scores['faithfulness'] < self.FAITHFULNESS_THRESHOLD:
            failures.append(
                f"Faithfulness score {scores['faithfulness']:.2f} < "
                f"{self.FAITHFULNESS_THRESHOLD}"
            )
        
        if scores['relevance'] < self.RELEVANCE_THRESHOLD:
            failures.append(
                f"Relevance score {scores['relevance']:.2f} < "
                f"{self.RELEVANCE_THRESHOLD}"
            )
        
        if scores['hallucination'] > self.HALLUCINATION_THRESHOLD:
            failures.append(
                f"Hallucination score {scores['hallucination']:.2f} > "
                f"{self.HALLUCINATION_THRESHOLD}"
            )
        
        if scores['citation_coverage'] < self.CITATION_COVERAGE_THRESHOLD:
            failures.append(
                f"Citation coverage {scores['citation_coverage']:.2f} < "
                f"{self.CITATION_COVERAGE_THRESHOLD}"
            )
        
        if scores['overall_quality'] < self.OVERALL_QUALITY_THRESHOLD:
            failures.append(
                f"Overall quality {scores['overall_quality']:.2f} < "
                f"{self.OVERALL_QUALITY_THRESHOLD}"
            )
        
        passed = len(failures) == 0
        return passed, failures
```

Ketika answer gagal quality gates:
- User ditampilkan error message: "Maaf, jawaban yang dihasilkan tidak memenuhi standar kualitas"
- Answer tidak ditampilkan ke user
- Failure reasons di-log untuk analysis
- Optionally: Retry dengan parameter yang berbeda atau suggest user untuk refinement query
