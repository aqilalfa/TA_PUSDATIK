# BAB 4.3: MODEL LANGUAGE MODEL DAN STRATEGI PROMPTING

## 4.3.1 Pemilihan Model Bahasa

Program ini tidak mengunci satu model tunggal secara statis, tetapi mengambil model aktif dari layanan model lokal yang disimpan sebagai default persisten. Jika model yang tersimpan tidak tersedia, sistem memiliki fallback ke model lain yang masih ada di Ollama. Namun, konfigurasi lingkungan tetap mengarah pada keluarga model Qwen2.5 dengan berkas GGUF yang disiapkan pada direktori model lokal.

Pemilihan model lokal dilakukan agar inference dapat berjalan di lingkungan internal tanpa bergantung pada API eksternal. Parameter model pada konfigurasi juga menunjukkan bahwa sistem diarahkan untuk bekerja dengan konteks yang cukup besar, jumlah token prediksi yang memadai, dan temperatur rendah agar jawaban stabil.

## 4.3.2 Jalur Prompt ke LLM

### 4.3.2.1 Penyusunan pesan ke Ollama

Pada lapisan runtime, fungsi `stream_answer()` membangun daftar pesan untuk endpoint `api/chat` milik Ollama. Urutan pesannya adalah sistem prompt, konteks referensi, riwayat percakapan, lalu pertanyaan pengguna beserta instruksi guardrail tambahan. Artinya, model tidak menerima prompt few-shot statis seperti pada rancangan konseptual awal, melainkan prompt berbasis konteks yang dibentuk dari hasil retrieval aktual.

### 4.3.2.2 Pemilihan prompt berdasarkan jenis query

Sistem membedakan `table`, `pasal`, `indikator`, dan `general` ketika memilih prompt yang akan dipakai. Query bertipe tabel memakai prompt yang menekankan pembacaan baris dan kolom secara literal. Query bertipe pasal dan indikator memakai prompt hukum yang menuntut kutipan presisi dan larangan mengarang nomor pasal atau ayat. Query umum memakai prompt general yang lebih fleksibel tetapi tetap harus grounded pada konteks yang tersedia.

## 4.3.3 Format Prompt yang Benar-benar Dipakai

### 4.3.3.1 System prompt berbasis aturan

Isi prompt yang aktif lebih dekat ke aturan berbasis kepatuhan daripada skema chain-of-thought eksplisit. Prompt menekankan agar model tidak mengarang nomor, tidak menambah interpretasi di luar dokumen, dan menulis jawaban dalam bahasa Indonesia formal. Untuk kasus tabel, prompt secara khusus memerintahkan model agar tidak mengubah nilai numerik maupun melewatkan baris yang tersedia di konteks.

### 4.3.3.2 Guardrail tekstual di level input model

Selain system prompt, fungsi streaming juga menambahkan guardrail tekstual ke isi pesan pengguna jika konteks dan query menunjukkan kebutuhan khusus. Ini bukan validator terpisah yang memblokir jawaban, melainkan instruksi tambahan yang dirangkai bersama pertanyaan agar model lebih fokus pada inti informasi yang ditemukan.

## 4.3.4 Pengaturan Inferensi Lokal

### 4.3.4.1 Parameter utama

Runtime Ollama menggunakan temperatur rendah, jumlah prediksi maksimal, dan context window besar agar beberapa dokumen, riwayat percakapan, dan prompt sistem masih dapat masuk ke dalam satu permintaan. Nilai context window pada implementasi streaming bahkan dinaikkan agar cukup untuk beberapa dokumen konteks dan history percakapan.

### 4.3.4.2 Deteksi model berpikir

Kode streaming juga memeriksa apakah model yang aktif termasuk keluarga `qwen3`, `qwen3.5`, atau `r1` dan kemudian menonaktifkan mode berpikir bila perlu. Fitur ini menunjukkan bahwa sistem memang disiapkan untuk beberapa model lokal berbeda, bukan hanya satu model tetap.

## 4.3.5 Alasan Arsitektural

Pendekatan prompt pada program ini menempatkan retrieval sebagai sumber utama kebenaran, sedangkan model bahasa hanya bertugas merangkai jawaban dari konteks yang telah dibatasi. Oleh karena itu, yang paling penting bukanlah prompt few-shot yang panjang, melainkan prompt yang konsisten, terbatas, dan cocok untuk dokumen legal atau tabel SPBE.

## 4.3.6 Ringkasan

Subbab ini menunjukkan bahwa jalur generasi pada program saat ini berjalan melalui pemilihan model lokal, penyusunan pesan ke Ollama, prompt berbasis aturan, dan streaming token langsung ke frontend. Penjelasan ini lebih sesuai dengan implementasi nyata dibanding narasi generik tentang model chat yang terlalu menekankan few-shot dan chain-of-thought.

## 4.3.7 Cuplikan Implementasi Streaming LLM

Berikut cuplikan fungsi streaming yang benar-benar dipakai oleh program. Fungsi ini membangun pesan ke Ollama, menambahkan guardrail tekstual, lalu mengalirkan token jawaban satu per satu ke caller.

```python
async def stream_answer(
	query: str,
	context: str,
	history: List,
	model_name: str,
	query_type: str = "general"
) -> AsyncIterator[str]:
	"""Stream LLM answer token by token via direct Ollama /api/chat call."""
	_PROMPT_MAP = {
		"table": SYSTEM_PROMPT_TABLE,
		"pasal": SYSTEM_PROMPT_LEGAL,
		"indikator": SYSTEM_PROMPT_LEGAL,
		"general": SYSTEM_PROMPT_GENERAL,
	}

	system_prompt = _PROMPT_MAP.get(query_type, SYSTEM_PROMPT_GENERAL)
	system_content = system_prompt + "\n\nKonteks Referensi:\n" + context

	ollama_messages = [{"role": "system", "content": system_content}]
	for msg in history:
		ollama_messages.append({"role": _role(msg), "content": msg.content})

	quality_guardrail = build_quality_guardrail(query, context)
	user_content = f"Pertanyaan: {query}"
	if quality_guardrail:
		user_content += f"\n\n{quality_guardrail}"
	ollama_messages.append({"role": "user", "content": user_content})

	options: dict = {
		"temperature": 0.1,
		"num_predict": 1024,
		"num_ctx": 8192
	}
```

Cuplikan ini menegaskan bahwa model bahasa pada program bukan berdiri sendiri, melainkan menerima konteks yang sudah disaring, jenis query yang sudah dikenali, dan riwayat percakapan yang relevan.
