# BAB 4.4: PIPELINE INGESTION, CHUNKING, DAN INDEXING DOKUMEN

## 4.4.1 Strategi Ingestion Dokumen

### 4.4.1.1 Document Type Classification

Sistem mengklasifikasikan dokumen input ke dalam tipe-tipe berikut:

1. **Peraturan** (Peraturan BSSN, Perpres, Permen)
   - Karakteristik: Struktur bab-pasal-ayat hirarkis
   - Chunk size: 900 karakter max
   - Overlap: 150 karakter
   - Metadata: Pasal number, Ayat number, Bab title

2. **Laporan Audit/Evaluasi** 
   - Karakteristik: Struktur paragraf dengan tabel data
   - Chunk size: 1800 karakter max
   - Overlap: 200 karakter
   - Metadata: Section, Subsection, Table references

3. **Pedoman Teknis**
   - Karakteristik: Panduan step-by-step dengan subbagian
   - Chunk size: 900-1200 karakter
   - Overlap: 150 karakter
   - Metadata: Step number, Chapter, Subsection

4. **Surat Edaran/Guidance**
   - Karakteristik: Directive documents dengan actionable items
   - Chunk size: 600-900 karakter
   - Overlap: 100 karakter
   - Metadata: Point number, Category

Klasifikasi otomatis berdasarkan:
- File naming convention
- Header/title patterns
- Structure detection (presence of "Pasal", "Bab", "Tabel", etc.)

### 4.4.1.2 OCR Processing Pipeline

Untuk dokumen scan/image-based:

```
Input PDF (scan)
    ↓
[PaddleOCR Processing] → Extract text from images
    ↓
[Language Detection] → Confirm Indonesian
    ↓
[Formatting Correction] → Fix OCR errors (common mistakes)
    ↓
[Layout Preservation] → Maintain table/list structures
    ↓
Output: Clean text ready for chunking
```

OCR menggunakan PaddleOCR dengan:
- Language: Indonesian
- GPU acceleration (NVIDIA CUDA)
- Confidence threshold: 0.7+ for text inclusion
- Post-processing: Regex-based correction for common OCR patterns

## 4.4.2 Chunking Strategies by Document Type

### 4.4.2.1 Chunking untuk Peraturan

Strategi khusus untuk mempertahankan legal structure:

```python
def chunk_peraturan(content: str, metadata: Dict) -> List[Document]:
    """
    Chunk peraturan per pasal/ayat untuk mempertahankan legal structure.
    Setiap chunk adalah complete pasal + ayatnya.
    """
    
    chunks = []
    pasal_pattern = re.compile(
        r"Pasal\s+(\d+)\s*(.*?)(?=Pasal|\Z)", 
        re.DOTALL|re.IGNORECASE
    )
    
    for match in pasal_pattern.finditer(content):
        pasal_num = match.group(1)
        pasal_content = match.group(2)
        
        # Split by Ayat if exists
        ayat_pattern = re.compile(
            r"Ayat\s+(\d+)\s*(.*?)(?=Ayat|\Z)",
            re.DOTALL|re.IGNORECASE
        )
        
        ayat_matches = list(ayat_pattern.finditer(pasal_content))
        
        if ayat_matches:
            # Split per ayat
            for ayat_match in ayat_matches:
                ayat_num = ayat_match.group(1)
                ayat_content = ayat_match.group(2)
                
                chunk_text = f"Pasal {pasal_num} Ayat {ayat_num}:\n{ayat_content}"
                
                # If too long, split with overlap
                if len(chunk_text) > MAX_CHUNK_SIZE_PERATURAN:
                    sub_chunks = split_with_overlap(
                        chunk_text,
                        max_size=MAX_CHUNK_SIZE_PERATURAN,
                        overlap=CHUNK_OVERLAP_PERATURAN
                    )
                    chunks.extend(sub_chunks)
                else:
                    chunks.append(chunk_text)
        else:
            # No Ayat structure, treat whole Pasal as chunk
            if len(pasal_content) > MAX_CHUNK_SIZE_PERATURAN:
                sub_chunks = split_with_overlap(...)
                chunks.extend(sub_chunks)
            else:
                chunks.append(f"Pasal {pasal_num}:\n{pasal_content}")
    
    # Attach metadata
    for i, chunk in enumerate(chunks):
        chunks[i] = Document(
            page_content=chunk,
            metadata={
                **metadata,
                'chunk_index': i,
                'chunk_type': 'pasal_ayat',
                'document_type': 'peraturan'
            }
        )
    
    return chunks
```

Keuntungan strategi ini:
- Maintains legal structure (pasal/ayat tetap utuh)
- Facilitates citation (mudah mereferensi Pasal X Ayat Y)
- Preserves legal semantics (konteks hukum tetap jelas)

### 4.4.2.2 Chunking untuk Laporan dengan Tabel

Laporan sering mengandung tabel yang penting untuk preservasi:

```python
def chunk_laporan_with_tables(content: str, metadata: Dict) -> List[Document]:
    """
    Chunk laporan sambil preserving table integrity.
    Tabel dijaga utuh, paragraf di-chunk dengan overlap.
    """
    
    chunks = []
    
    # Extract tabel terlebih dahulu
    table_pattern = re.compile(
        r"(?:Tabel|TABLE)\s+(\d+).*?(?=Tabel|TABLE|$)",
        re.DOTALL|re.IGNORECASE
    )
    
    # Paragraph chunking dengan table preservation
    parts = table_pattern.split(content)
    
    for part in parts:
        # If part is table, keep whole
        if is_table_like_text(part):
            chunks.append(part)
        else:
            # Chunk paragraphs dengan overlap
            sub_chunks = split_with_overlap(
                part,
                max_size=MAX_CHUNK_SIZE_LAPORAN,
                overlap=CHUNK_OVERLAP_LAPORAN
            )
            chunks.extend(sub_chunks)
    
    # Attach metadata
    for i, chunk in enumerate(chunks):
        chunks[i] = Document(
            page_content=chunk,
            metadata={
                **metadata,
                'chunk_index': i,
                'chunk_type': 'paragraph_or_table',
                'document_type': 'laporan',
                'contains_table': is_table_like_text(chunk)
            }
        )
    
    return chunks
```

## 4.4.3 Metadata Extraction dan Structuring

Setiap chunk dilengkapi dengan metadata terstruktur:

```python
class ChunkMetadata(BaseModel):
    """Metadata untuk setiap chunk."""
    
    # Document-level
    document_id: int                    # Database reference
    document_title: str                 # Full document title
    document_type: str                  # Type: peraturan, laporan, etc.
    source_file: str                    # Original filename
    ingestion_date: datetime            # When document was indexed
    
    # Chunk-level
    chunk_index: int                    # Sequential chunk number
    chunk_type: str                     # pasal_ayat, paragraph, table, etc.
    
    # Structural
    hierarchy: Dict[str, str]           # {'bab': '2', 'pasal': '5', 'ayat': '3'}
    section_title: Optional[str]        # Section/subsection name
    
    # Content markers
    contains_table: bool                # Whether chunk has table data
    contains_list: bool                 # Whether chunk has bullet/numbered list
    contains_definition: bool           # Whether chunk defines a term
    
    # Extraction metadata
    confidence_score: float             # OCR/extraction confidence
    is_reformatted: bool                # Whether formatting was corrected
```

Metadata ini digunakan untuk:
- **Better retrieval**: Filtering berdasarkan document type, hierarchy level
- **Citation generation**: Automatic reference building (e.g., "Pasal X Ayat Y")
- **Context preservation**: Understanding document structure
- **Quality tracking**: Monitoring OCR accuracy, formatting issues

## 4.4.4 Indexing ke Vector Store

Proses indexing ke Qdrant:

```python
async def index_chunks_to_qdrant(
    chunks: List[Document],
    embeddings_model,
    qdrant_client,
    collection_name: str
):
    """
    Index chunks ke Qdrant dengan batching untuk performance.
    """
    
    batch_size = 32
    
    for i in range(0, len(chunks), batch_size):
        batch = chunks[i:i+batch_size]
        
        # Generate embeddings
        texts = [doc.page_content for doc in batch]
        embeddings = await embeddings_model.embed_documents(texts)
        
        # Prepare points untuk Qdrant
        points = []
        for doc, embedding in zip(batch, embeddings):
            point = PointStruct(
                id=generate_point_id(),
                vector=embedding,
                payload={
                    'doc_id': doc.metadata['document_id'],
                    'chunk_index': doc.metadata['chunk_index'],
                    'title': doc.metadata['document_title'],
                    'type': doc.metadata['document_type'],
                    'hierarchy': doc.metadata.get('hierarchy', {}),
                    'content_preview': doc.page_content[:200]
                }
            )
            points.append(point)
        
        # Upsert ke Qdrant
        await qdrant_client.upsert(
            collection_name=collection_name,
            points=points
        )
        
        logger.info(f"Indexed {len(batch)} chunks to Qdrant")
```
