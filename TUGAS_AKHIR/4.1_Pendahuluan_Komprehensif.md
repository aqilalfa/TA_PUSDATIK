# BAB 4: HASIL PENELITIAN

## 4.1 Pendahuluan

Bab ini memaparkan hasil penelitian dari beberapa tahap penelitian yang telah dirancang dan dijelaskan pada subbab 3.3, mencakup tahapan *design and development*, *demonstration*, dan *evaluation*. Melalui serangkaian tahapan penelitian tersebut, telah dibangun sebuah sistem pertanyaan-jawaban berbasis teknologi kecerdasan buatan yang mengintegrasikan pendekatan *Retrieval-Augmented Generation* (RAG) dengan strategi keamanan berlapis menggunakan *guardrails*. Sistem yang dibangun dirancang khusus untuk memenuhi kebutuhan layanan informasi publik di Lembaga Sertifikasi Produk (LSPro) Teknologi Keamanan Siber dan Sandi (BSSN).

### Motivasi Penelitian dan Problem Statement

Institusi publik seperti LSPro memiliki tantangan signifikan dalam penyediaan layanan informasi berkualitas tinggi kepada stakeholder. Pertanyaan-pertanyaan yang masuk mencakup berbagai aspek teknis dan administratif terkait sertifikasi produk, regulasi, dan kebijakan internal. Menjawab setiap pertanyaan secara manual memerlukan sumber daya manusia yang besar dan kurang skalabel untuk volume pertanyaan yang terus meningkat.

Sebelum penelitian ini, LSPro tidak memiliki sistem otomatis untuk menjawab pertanyaan pengguna secara akurat dan konsisten berdasarkan dokumen-dokumen resmi yang telah ditetapkan. Hal ini mengakibatkan:
- **Keterlambatan respons**: Pengguna menunggu lama untuk mendapatkan jawaban akurat
- **Inkonsistensi informasi**: Jawaban yang diberikan berbeda tergantung pada petugas yang menangani
- **Beban operasional tinggi**: Tim LSPro menghabiskan waktu signifikan untuk menjawab pertanyaan yang sama berulang kali
- **Keterbatasan skalabilitas**: Tidak ada mekanisme otomatis untuk menangani pertanyaan dalam jumlah besar

### Solusi yang Diajukan: Sistem RAG Terintegrasi

Untuk mengatasi tantangan tersebut, penelitian ini mengajukan sebuah solusi sistem chatbot berbasis **Retrieval-Augmented Generation** yang mengintegrasikan:

1. **Knowledge Base Terstruktur**: Kumpulan dokumen-dokumen resmi LSPro yang diproses dan diindeks secara otomatis menggunakan kombinasi vector database (Qdrant) dan full-text search (BM25)

2. **RAG Pipeline Hibrida**: Sistem pencarian yang menggabungkan dense retrieval (berbasis embedding semantik) dan sparse retrieval (berbasis kata kunci) untuk mendapatkan dokumen paling relevan

3. **Reranking Bertingkat**: Mekanisme perangkingan ulang menggunakan cross-encoder untuk memastikan hanya konteks paling relevan yang dikirimkan ke LLM

4. **Prompt Engineering Khusus**: Sistem prompt yang dirancang untuk memandu LLM dalam menjawab pertanyaan secara akurat sesuai dengan domain hukum dan peraturan

5. **Guardrails Keamanan**: Lapisan validasi keamanan multi-tingkat yang mencegah LLM memberikan jawaban yang tidak etis, ofensif, atau mengandung jailbreak

6. **Quality Assurance Gates**: Sistem penilaian otomatis untuk mengukur kualitas jawaban sebelum ditampilkan kepada pengguna

### Struktur Implementasi

Penelitian ini mengimplementasikan seluruh stack teknologi yang diperlukan untuk mendukung sistem RAG production-ready di lingkungan internal LSPro. Stack teknologi mencakup:

- **Backend**: FastAPI (Python) sebagai API server yang menangani request dari pengguna
- **Embedding Model**: Sentence-BERT Indo (indo-sentence-bert-base) untuk menghasilkan representasi vektor semantik dokumen
- **Vector Database**: Qdrant sebagai penyimpan dan pencarian vektor embedding
- **BM25 Retriever**: Hybrid search integration dengan full-text search capability
- **LLM Local**: Qwen 2.5 7B dalam format GGUF Q4_K_M yang dijalankan melalui Ollama untuk inference lokal tanpa ketergantungan layanan eksternal
- **Frontend**: Vue.js 3 dengan Vite untuk user interface chatbot
- **Security**: Guardrails AI untuk jailbreak detection dan toxic language filtering

### Kontribusi Utama

Penelitian ini memberikan beberapa kontribusi penting:

1. **Sistem RAG Production-Ready**: Demonstrasi implementasi lengkap dari sistem RAG yang dapat digunakan dalam lingkungan produksi internal pemerintah dengan constraint sumber daya terbatas

2. **Prompt Engineering untuk Domain Hukum**: Pengembangan sistem prompt yang khusus dirancang untuk memastikan jawaban akurat pada dokumen peraturan dan hukum Indonesia

3. **Multi-Layer Security Architecture**: Implementasi keamanan berlapis yang menggabungkan input validation, jailbreak detection, output validation, dan quality gates

4. **Hybrid Retrieval Strategy**: Demonstrasi efektivitas kombinasi dense retrieval (semantic) dan sparse retrieval (keyword-based) untuk meningkatkan relevance dan recall

5. **Quality Assurance Framework**: Framework terukur untuk mengevaluasi kualitas RAG system menggunakan metrik RAGAS

### Fase Penelitian dan Deliverables

Penelitian ini diorganisir dalam tiga fase utama yang saling mendukung:

**Fase 1: Design and Development (~60% waktu penelitian)**
- Merancang arsitektur RAG system dengan komponen-komponen termodular
- Membangun knowledge base dari dokumen-dokumen LSPro
- Mengimplementasikan chatbot backend dengan FastAPI
- Mengembangkan frontend interface dengan Vue.js
- Mengintegrasikan guardrails dan quality gates

**Fase 2: Demonstration (~20% waktu penelitian)**
- Melakukan deployment di infrastruktur internal LSPro
- Simulasi skenario pengguna (end-user queries)
- Simulasi skenario administrator (document ingestion)
- Verifikasi bahwa seluruh sistem berfungsi sesuai rancangan

**Fase 3: Evaluation (~20% waktu penelitian)**
- Pengujian performa RAG menggunakan RAGAS metrics
- Security testing dengan jailbreak prompt injection
- User experience testing dengan BUS-11 questionnaire
- Dokumentasi hasil dan rekomendasi
